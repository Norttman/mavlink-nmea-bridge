# MAVLink ↔ NMEA Bridge

Мост между наземной станцией управления (**Mission Planner** или **QGroundControl**) и **Alpine Quest**. Транслирует телеметрию дрона (MAVLink) в формат NMEA 0183 по TCP, позволяя видеть позицию БПЛА на офлайн-картах Alpine Quest в реальном времени.

Написан на чистом C, без внешних зависимостей кроме стандартной библиотеки MAVLink. Собирается под Linux и Windows.

## 🎯 Зачем это нужно

- Видеть дрон на **офлайн-картах** Alpine Quest (топография, спутник, свои слои).
- Иметь **резервный канал отображения** позиции БПЛА, независимый от наземной станции.
- Работать в условиях, когда ГСН недоступна или неудобна (например, на планшете в поле).
- Интегрировать телеметрию дрона в любые приложения, принимающие NMEA по TCP/UDP.

## ⚙️ Как это работает

```
[Дрон] ──MAVLink──> [НСУ] ──кабели──> [Пульт с Wi-Fi]
                                               │
                                               │ Wi-Fi
                                               ▼
                                       [Ноутбук с ГСН]
                                       (MP или QGC)
                                               │
                                               │ UDP 127.0.0.1:14550
                                               ▼
                                       [nmea_bridge]
                                               │
                                               │ TCP порт 10000
                                               ▼
                                       [Телефон по USB-модему]
                                               │
                                               ▼
                                       [Alpine Quest]
```

Программа слушает UDP-поток MAVLink от наземной станции, парсит сообщения `GPS_RAW_INT`, конвертирует координаты в NMEA-предложения `$GPGGA` и `$GPRMC` и отдаёт их по TCP.

## 📋 Возможности

- ✅ Кроссплатформенность: Linux (нативно) и Windows (через MinGW или MSYS2)
- ✅ Совместимость с **Mission Planner** и **QGroundControl**
- ✅ Минимальный размер бинарника (~85 КБ)
- ✅ Нет зависимостей от Python, Java или .NET
- ✅ Передача координат, высоты, скорости и курса
- ✅ Автоматическая фильтрация по типу GPS-фикса (только 3D Fix)
- ✅ Корректный расчёт контрольной суммы NMEA

## 🖥️ Системные требования

**Для сборки:**
- GCC (любая современная версия)
- Make
- Git

**Для работы:**
- Mission Planner **или** QGroundControl с настроенным UDP-форвардингом
- Alpine Quest **Pro** (в бесплатной версии нет Network GPS)
- Связка: НСУ → пульт с Wi-Fi → ноутбук → телефон по USB-модему

## Установка и сборка

### Шаг 1. Клонирование репозитория

```bash
git clone https://github.com/ТВОЙ_ЮЗЕР/mavlink-nmea-bridge.git
cd mavlink-nmea-bridge
```

### Шаг 2. Загрузка заголовков MAVLink

Заголовки не лежат в репозитории (они большие). Скачай их отдельно:

```bash
git clone https://github.com/mavlink/c_library_v2.git mavlink
```

После этого структура должна быть:

```
mavlink-nmea-bridge/
├── src/main.c
├── Makefile
├── Makefile.win
└── mavlink/
    ├── common/
    │   └── mavlink.h
    └── ...
```

---

### Сборка под Linux

```bash
make
```

Появится бинарник `nmea_bridge`. Запуск:

```bash
./nmea_bridge
```

---

### Сборка под Windows

Есть два пути.

#### Путь А: Кросс-компиляция из Linux (быстро, 1 минута)

**Ubuntu/Debian:**

```bash
sudo apt install gcc-mingw-w64
```

**Fedora:**

```bash
sudo dnf install mingw64-gcc
```

**Сборка:**

```bash
x86_64-w64-mingw32-gcc -Wall -O2 -I. -Wno-format-truncation \
    -o nmea_bridge.exe src/main.c -lws2_32 -lm
```

Полученный `nmea_bridge.exe` можно запускать на любой Windows 10/11 без установки чего-либо.

#### Путь Б: Нативная сборка в Windows через MSYS2

1. Установи [MSYS2](https://www.msys2.org/) в `C:\msys64`.
2. Запусти **MSYS2 MinGW x64**.
3. Установи компилятор:

```bash
pacman -Syu
pacman -S mingw-w64-x86_64-gcc mingw-w64-x86_64-make
```

4. Собери проект:

```bash
mingw32-make -f Makefile.win
```

## Настройка Alpine Quest (подробно)

> Функция Network GPS доступна только в **Alpine Quest Pro** (платная версия). В бесплатной версии этого пункта меню нет.

### Шаг 1. Открой настройки

- Запусти Alpine Quest на телефоне.
- Нажми на иконку меню (три горизонтальные полоски в левом верхнем углу).
- Выбери **Settings** (Настройки).

### Шаг 2. Перейди в раздел GPS

- В списке настроек найди и нажми **GPS**.
- Откроется экран с параметрами GPS.

### Шаг 3. Смени источник GPS

- Найди пункт **GPS source** (Источник GPS) или **GPS device**.
- По умолчанию там стоит **Internal GPS** (встроенный GPS телефона).
- Нажми на него и выбери **Network (TCP/UDP) GPS device** (Сетевое GPS-устройство).

### Шаг 4. Настрой параметры подключения

После выбора Network GPS появятся дополнительные поля:

| Параметр | Значение |
|----------|----------|
| **Protocol** (Протокол) | `TCP` |
| **IP address** (IP-адрес) | IP ноутбука в сети USB-модема (обычно `192.168.42.100`) |
| **Port** (Порт) | `10000` |
| **Auto-reconnect** (Автопереподключение) | Включить |

**Как узнать IP ноутбука:**
- На ноутбуке открой `cmd` (Windows) или терминал (Linux).
- Выполни `ipconfig` (Windows) или `ip addr` (Linux).
- Найди адаптер **Remote NDIS based Internet Sharing Device** (Windows) или `usb0` / `rndis0` (Linux).
- IP будет из подсети `192.168.42.x` — это и есть нужный адрес.

### Шаг 5. Сохрани и проверь

- Нажми **Save** (Сохранить) или вернись назад (настройки сохраняются автоматически).
- Вернись на главный экран карты.

### Шаг 6. Проверь статус подключения

- В верхней части экрана (рядом с иконкой спутников) должна появиться надпись **Network** или **TCP** вместо **Internal**.
- Через 2-3 секунды на карте появится **вторая иконка** (обычно синяя или зелёная точка) — это позиция дрона из наземной станции.
- Если дрон стоит на месте — точка тоже стоит. Если дрон двигается — точка повторяет его траекторию.

### Шаг 7. Настройка внешнего вида (опционально)

Чтобы иконка дрона отличалась от твоей позиции:
- **Меню → Settings → Map display → GPS markers**.
- Найди пункт **Network GPS marker** (или аналогичный).
- Выбери другой цвет или иконку (например, красную точку или символ самолёта).

## Настройка наземной станции

Наземная станция должна форвардить MAVLink-поток на UDP-порт `14550`. Ниже инструкции для Mission Planner и QGroundControl.

### Вариант 1: Mission Planner

**Способ А — через скрытое меню (быстро):**

1. Запусти Mission Planner и подключись к дрону.
2. Нажми **`Ctrl + F`** → откроется окно `Temp`.
3. Найди раздел **UDP Forward** / **MAVLink Forwarding**.
4. Добавь адрес: `127.0.0.1:14550`.

**Способ Б — через конфиг:**

В MP зайди в **ConfigTuner → Full List → Planner → MAVLink** и проверь, что `udpout` установлен на `127.0.0.1:14550`.

---

### Вариант 2: QGroundControl

QGroundControl — кроссплатформенная наземная станция (Windows, Linux, macOS, Android).

**Способ А — через GUI:**

1. Запусти QGroundControl.
2. Нажми на иконку **Q** в левом верхнем углу → **Application Settings**.
3. Перейди во вкладку **MAVLink** (или **General → MAVLink**).
4. В разделе **MAVLink Forwarding** добавь:
   - **Host:** `127.0.0.1`
   - **Port:** `14550`

**Способ Б — через командную строку:**

**Windows:**

```cmd
qgroundcontrol.exe --mavlink-output 127.0.0.1:14550
```

**Linux:**

```bash
./qgroundcontrol --mavlink-output 127.0.0.1:14550
```

**macOS:**

```bash
open -a QGroundControl --args --mavlink-output 127.0.0.1:14550
```

> **Совет:** если в твоей версии QGC нет пункта **MAVLink Forwarding** в GUI, используй запуск с аргументом `--mavlink-output`. Это работает во всех версиях QGC.

## 🎮 Сценарий использования: НСУ + пульт + ноутбук + телефон

Это основной и единственный поддерживаемый сценарий.

### Физическая схема

```
[Дрон] ↔ [НСУ: 2 антенны] ──кабели──> [Пульт управления]
                                               │
                                         [Wi-Fi точка]
                                               │
                                               │ Wi-Fi
                                               ▼
[Телефон с AQ] ──USB-кабель──> [Ноутбук] ←──Wi-Fi──┘
              (USB-модем)
```

### Пошаговый запуск

**1. Подключи ноутбук к Wi-Fi пульта НСУ**
- Найди Wi-Fi сеть пульта (обычно `NSU_XXXX` или `Pilot_XXXX`).
- Подключись (пароль обычно `12345678` или написан на пульте).
- Убедись, что ноутбук получил IP из подсети пульта (обычно `192.168.1.x`).

**2. Подключи телефон к ноутбуку по USB**
- На телефоне: **Настройки → Точка доступа и модем → USB-модем** (включи).
- На ноутбуке появится новый сетевой адаптер с IP из подсети `192.168.42.x`.
- Запиши этот IP — он нужен для Alpine Quest.

**3. Запусти наземную станцию (Mission Planner или QGroundControl)**
- Подключись к дрону через Wi-Fi пульта (обычно `UDPCL:192.168.1.1:14550` или `TCP:192.168.1.1:5760`).
- Убедись, что телеметрия идёт и GPS имеет **3D Fix**.

**4. Настрой форвардинг ГСН** (см. раздел выше).

**5. Запусти `nmea_bridge`**
- **Linux:** `./nmea_bridge`
- **Windows:** `nmea_bridge.exe` (при первом запуске Firewall спросит разрешение — разреши для **частных сетей**)
- В консоли должно появиться:

```
=== MAVLink to NMEA Bridge ===
Listening MAVLink on UDP:14550
TCP server on port 10000. Waiting for Alpine Quest...
```

**6. Настрой Alpine Quest** (см. раздел выше).
- IP: `192.168.42.100` (IP ноутбука в USB-сети)
- Порт: `10000`

**7. Проверь результат**
- В консоли `nmea_bridge` появится: `Alpine Quest connected! IP: 192.168.42.x`
- На карте телефона появится точка дрона.

## Troubleshooting

### Alpine Quest не подключается

- Проверь **Firewall Windows** — порт 10000 должен быть открыт для входящих TCP-соединений.
- Убедись, что USB-модем действительно поднят (`ipconfig` показывает адаптер Remote NDIS).
- Попробуй пинг с телефона: `ping 192.168.42.100`.

### nmea_bridge не видит MAVLink

- Убедись, что наземная станция запущена и подключена к дрону.
- Проверь, что форвардинг настроен на порт 14550.
- В Linux проверь, что порт не занят: `ss -ulnp | grep 14550`.

### Точка не появляется на карте

- Дрон должен иметь **3D GPS Fix** (иначе `nmea_bridge` не отправляет NMEA).
- Проверь консоль `nmea_bridge` — должна быть строка `Alpine Quest connected!`.
- В Alpine Quest убедись, что выбран именно **Network GPS**, а не Internal.

### Ошибка компиляции: `mavlink/common/mavlink.h: No such file`

- Используй флаг `-I.` (точка), а не `-Imavlink`.
- Убедись, что папка `mavlink/common/mavlink.h` действительно существует.

### Wi-Fi пульта нестабильный

- Ноутбук может терять связь с дроном. В этом случае следи за RSSI в ГСН.
- Связь ноутбук-телефон по USB останется стабильной.
- Если связь с дроном потеряна — дрон должен вернуться по RTH.
